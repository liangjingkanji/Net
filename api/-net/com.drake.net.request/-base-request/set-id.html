请求ID Group和Id在使用场景上有所区别, 预期上Group允许重复赋值给多个请求, Id仅允许赋值给一个请求, 但实际上都允许重复赋值 在作用域中发起请求时会默认使用协程错误处理器作为Group: setGroup(coroutineContext[CoroutineExceptionHandler]) 如果你覆盖Group会导致协程结束不会自动取消请求
